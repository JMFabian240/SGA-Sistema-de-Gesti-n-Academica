# RF-45: Modificar tipos de beca

| Campo | Descripción |
| :---: | ----- |
| **Autor** | José Manuel Fabian Hernández |
| **Nombre** | RF-45: Modificar tipos de beca |
| **Actor** | Administrador y Gestor Administrativo |
| **Objetivo** | Actualizar los atributos de un tipo de beca ya registrado en el catálogo (criterio de asignación, porcentaje u otro campo), reflejando los cambios de forma inmediata para los procesos de inscripción futuros sin alterar las becas ya asignadas en ciclos activos. |
| **Flujo Principal** | 1. El actor (Administrador o Gestor) ingresa al módulo de Gestión de Becas.<br>2. Selecciona la opción "Consultar catálogo de becas" y localiza el tipo de beca que desea modificar.<br>3. El actor hace clic en "Editar" sobre el tipo de beca seleccionado.<br>4. El sistema habilita los campos del formulario con los datos actuales precargados.<br>5. El actor actualiza el criterio de asignación, el porcentaje de descuento u otro atributo permitido.<br>6. El actor hace clic en "Guardar cambios".<br>7. El sistema valida que los nuevos datos sean correctos y que no se duplique el nombre con otra beca existente en el catálogo.<br>8. El sistema actualiza el catálogo con los nuevos valores y muestra el mensaje: "Tipo de beca actualizado correctamente". |
| **Flujo Alterno** | <b>A.</b> Cancelación de la modificación:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 5, el actor decide no aplicar cambios y hace clic en "Cancelar".<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema descarta los cambios y regresa a la vista del catálogo sin alterar los datos.<br><br><b>B.</b> Nombre de beca duplicado al modificar:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 7, el sistema detecta que el nuevo nombre ingresado ya corresponde a otro tipo de beca existente en el catálogo.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra el mensaje: "Ya existe un tipo de beca con ese nombre. Ingrese uno diferente".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor corrige el nombre y vuelve a intentar guardar. |
| **Precondiciones** | El actor debe tener sesión activa. El tipo de beca a modificar debe existir previamente en el catálogo del sistema. |
| **Postcondiciones** | El catálogo de becas queda actualizado con los nuevos valores. Los cambios aplican únicamente para asignaciones futuras; las becas ya asignadas a alumnos en ciclos activos no se ven modificadas retroactivamente. |
| **Reglas de negocio involucradas** | • La modificación del catálogo de becas está habilitada para los perfiles Administrador y Gestor Administrativo.<br>• Los cambios en el catálogo aplican exclusivamente para futuros procesos de inscripción; no se modifican retroactivamente las becas ya asignadas en ciclos activos.<br>• El porcentaje de descuento modificado debe mantenerse dentro del rango válido de 1 a 100\. |
