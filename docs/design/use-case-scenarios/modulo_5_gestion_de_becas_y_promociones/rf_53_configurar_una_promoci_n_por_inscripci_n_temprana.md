# RF-53: Configurar una promoción por inscripción temprana

| Campo | Descripción |
| :---: | ----- |
| **Autor** | Diana Jareth Ritt Martínez |
| **Nombre** | RF-53: Configurar una promoción por inscripción temprana |
| **Actor** | Administrador |
| **Objetivo** | Permitir usuario administrador configurar un periodo de promoción por inscripción temprana, indicando la fecha de inicio, la fecha de finalización y el beneficio que se aplicará, para que el sistema otorgue automáticamente el descuento o la beca correspondiente a los alumnos que realicen el pago de inscripción dentro del periodo establecido. |
| **Flujo Principal** | 1. El Administrador accede al módulo de ciclo escolar.<br>2. El sistema muestra las opciones disponibles.<br>3. El Administrador selecciona la opción Promoción por inscripción temprana.<br>4. El sistema muestra el formulario de configuración.<br>5. El Administrador selecciona el ciclo escolar correspondiente.<br>6. El Administrador ingresa la fecha de inicio y la fecha de finalización.<br>7. El Administrador selecciona el tipo de beneficio.<br>8. El Administrador define el porcentaje de descuento o la beca correspondiente.<br>9. El sistema valida las fechas y el beneficio seleccionado.<br>10. El sistema muestra un resumen de la configuración.<br>11. El Administrador revisa la información y selecciona Guardar promoción.<br>12. El sistema solicita confirmación.<br>13. El Administrador confirma la operación.<br>14. El sistema registra la promoción y la activa para el periodo configurado.<br>15. El sistema registra la operación en la bitácora y muestra un mensaje de éxito. |
| **Flujo Alterno** | Flujo alterno A — La fecha final es anterior a la fecha inicial<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El Administrador ingresa las fechas de la promoción.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema detecta que la fecha de finalización es anterior a la fecha de inicio.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema informa que el periodo no es válido.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema no permite guardar la promoción.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El Administrador corrige una o ambas fechas.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El sistema vuelve a validar la información.<br>&nbsp;&nbsp;&nbsp;&nbsp;7. El flujo continúa desde el paso 14 del flujo principal. Flujo alterno B — No se seleccionó un beneficio<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El Administrador intenta guardar la promoción.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema detecta que no se seleccionó un porcentaje ni un tipo de beca.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema informa que es obligatorio definir el beneficio.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema no guarda la promoción.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El Administrador selecciona el beneficio correspondiente.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El flujo continúa desde el paso 15 del flujo principal. Flujo alterno C — El porcentaje de descuento no es válido<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El Administrador selecciona la opción de porcentaje de descuento.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El Administrador ingresa un valor igual a cero, negativo o mayor que el permitido.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema informa que el porcentaje no es válido.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema solicita corregir el dato.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El Administrador ingresa un porcentaje válido.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El sistema vuelve a validar la información. Flujo alterno D — No existen tipos de beca disponibles<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El Administrador selecciona la opción de tipo de beca.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema consulta el catálogo de becas.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema detecta que no existen becas disponibles.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema informa que no puede seleccionarse ese tipo de beneficio.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El Administrador puede registrar previamente un tipo de beca o seleccionar un porcentaje de descuento.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. La promoción no se guarda hasta que se defina un beneficio válido. Flujo alterno E — Ya existe una promoción en el mismo periodo<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El Administrador ingresa las fechas de la promoción.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema detecta que ya existe una promoción activa para el mismo ciclo y periodo.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema informa que existe una coincidencia.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema muestra los datos de la promoción existente.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El Administrador puede modificar las fechas de la nueva promoción.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El sistema no permite guardar dos promociones incompatibles para el mismo periodo. Flujo alterno F — El pago se registra fuera del periodo promocional<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario registra un pago de inscripción.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema consulta la fecha del pago.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema detecta que la fecha no se encuentra dentro del periodo promocional.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema no aplica el beneficio.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El sistema conserva el monto regular de la inscripción.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El pago continúa con el procedimiento normal. |
| **Precondiciones** | • El Administrador debe haber iniciado sesión. • El usuario debe tener permisos para configurar promociones. • Debe existir un ciclo escolar registrado. • El ciclo escolar relacionado con la promoción debe estar disponible para inscripciones. • Deben existir tipos de beca o porcentajes de descuento que puedan utilizarse como beneficio. • La promoción no debe estar registrada previamente con los mismos datos. |
| **Postcondiciones** | **Si la promoción se configura correctamente:** 1\. 	La promoción queda relacionada con el ciclo escolar seleccionado. 2\. 	El periodo de inicio y finalización queda registrado. 3\. 	El beneficio queda definido mediante un porcentaje o tipo de beca. 4\. 	El sistema puede aplicar automáticamente el beneficio a los alumnos que cumplan las condiciones. 5\. 	La configuración queda registrada en la bitácora. **Si un alumno cumple las condiciones de la promoción:** 1\. 	El beneficio se aplica automáticamente. 2\. 	El monto de inscripción queda actualizado. 3\. 	El beneficio queda registrado en el expediente financiero del alumno. 4\. 	El beneficio no puede aplicarse dos veces al mismo alumno. |
| **Reglas de negocio involucradas** | • Solo el Administrador puede configurar promociones por inscripción temprana.<br>• La promoción debe estar relacionada con un ciclo escolar.<br>• La fecha de finalización no puede ser anterior a la fecha de inicio.<br>• El beneficio debe consistir en un porcentaje de descuento o en un tipo de beca.<br>• El porcentaje de descuento debe encontrarse dentro del rango permitido.<br>• El tipo de beca debe existir previamente en el catálogo.<br>• El beneficio solo debe aplicarse a pagos de inscripción realizados dentro del periodo establecido.<br>• El beneficio debe aplicarse automáticamente cuando el alumno cumpla las condiciones.<br>• El mismo beneficio no debe aplicarse más de una vez al mismo alumno.<br>• Los pagos realizados fuera del periodo deben conservar el monto regular.<br>• La promoción deja de aplicarse cuando finaliza el periodo configurado.<br>• Toda configuración debe quedar registrada en la bitácora de auditoría. |
