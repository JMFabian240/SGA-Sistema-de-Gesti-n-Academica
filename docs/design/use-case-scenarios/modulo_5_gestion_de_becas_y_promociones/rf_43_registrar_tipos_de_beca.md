# RF-43: Registrar tipos de beca

| Campo | Descripción |
| :---: | ----- |
| **Autor** | José Manuel Fabian Hernández |
| **Nombre** | RF-43: Registrar tipos de beca |
| **Actor** | Administrador y Gestor Administrativo |
| **Objetivo** | Crear y registrar en el catálogo del sistema los tipos de beca disponibles en la institución, especificando su nombre, criterio de asignación y porcentaje de descuento, para su uso en los procesos de inscripción del ciclo escolar. |
| **Flujo Principal** | 1. El actor (Administrador o Gestor) ingresa al módulo de Gestión de Becas.<br>2. Selecciona la opción "Registrar nuevo tipo de beca".<br>3. El sistema despliega el formulario de registro con los campos: nombre de la beca, criterio de asignación y porcentaje de descuento.<br>4. El actor ingresa el nombre de la beca (ej. "Beca por hermanos", "Beca por calificación").<br>5. El actor selecciona el criterio de asignación entre las opciones disponibles: tiempo de inscripción, calificación o beca por hermanos.<br>6. El actor ingresa el porcentaje de descuento que aplica la beca.<br>7. El actor hace clic en "Guardar".<br>8. El sistema valida que todos los campos sean correctos y que no exista una beca con el mismo nombre en el catálogo.<br>9. El sistema almacena el nuevo tipo de beca y muestra el mensaje: "Tipo de beca registrado correctamente". |
| **Flujo Alterno** | <b>A.</b> Nombre de beca duplicado:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 8, el sistema detecta que ya existe un tipo de beca con el mismo nombre en el catálogo.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra el mensaje: "Ya existe un tipo de beca con ese nombre. Ingrese un nombre diferente".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor modifica el nombre y vuelve a intentar guardar.<br><br><b>B.</b> Campos incompletos o porcentaje inválido:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 8, el sistema detecta que el porcentaje de descuento está vacío o fuera del rango permitido.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema resalta el campo faltante en rojo y muestra: "Todos los campos son obligatorios. El porcentaje debe ser un valor entre 1 y 100".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor completa o corrige la información y reintenta. |
| **Precondiciones** | El actor debe tener sesión activa con los privilegios de Administrador o Gestor Administrativo. |
| **Postcondiciones** | El nuevo tipo de beca queda disponible en el catálogo del sistema para ser asignada a alumnos durante el proceso de inscripción del ciclo escolar activo y futuros. |
| **Reglas de negocio involucradas** | • El registro de tipos de beca está habilitado para los perfiles Administrador y Gestor Administrativo.<br>• Los criterios de asignación permitidos son exclusivamente: tiempo de inscripción, calificación o beca por hermanos.<br>• El porcentaje de descuento debe ser un valor numérico entre 1 y 100\. No se permiten valores fuera de este rango. |
