# RF-63: Registrar un pago adelantado de colegiaturas

| Campo | Descripción |
| :---: | ----- |
| **Autor** | Diana Jareth Ritt Martinez |
| **Nombre** | RF-63: Registrar un pago adelantado de colegiaturas |
| **Actor** | Administrador o Gestor Administrativos |
| **Objetivo** | Permitir al personal administrativo registrar el pago anticipado de varias colegiaturas futuras, indicando el número de meses, el monto total y el método de pago, para marcar los periodos correspondientes como pagados y evitar la generación de recargos o alertas de morosidad. |
| **Flujo Principal** | 1. El usuario accede al módulo de pagos.<br>2. Busca y selecciona al alumno.<br>3. El sistema muestra el calendario de colegiaturas pendientes.<br>4. El usuario selecciona la opción **Registrar pago adelantado**.<br>5. Indica la cantidad de meses a cubrir.<br>6. El usuario ingresa o confirma el monto, método y fecha de pago.<br>7. El sistema muestra un resumen de la operación.<br>8. El usuario confirma el registro.<br>9. El sistema registra el pago adelantado.<br>10. El sistema marca los periodos seleccionados como **Pagados**.<br>11. El sistema actualiza el calendario y muestra un mensaje de éxito. |
| **Flujo Alterno** | A — El monto no corresponde con los meses seleccionados<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario indica el número de meses.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema calcula el total correspondiente.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El usuario ingresa un monto diferente.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema detecta que el importe no coincide.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El sistema muestra el monto correcto.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El sistema solicita corregir el dato.<br>&nbsp;&nbsp;&nbsp;&nbsp;7. El usuario ingresa el monto correspondiente.<br>&nbsp;&nbsp;&nbsp;&nbsp;8. El flujo continúa desde la validación de la operación. B — Algunos meses ya están pagados<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema consulta el calendario.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema identifica uno o más meses ya liquidados.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema excluye esos periodos.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema muestra únicamente los meses pendientes.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El usuario selecciona la cantidad de meses disponibles.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El sistema calcula el monto utilizando solo los periodos pendientes. C — No existen meses futuros pendientes<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario selecciona la opción de pago adelantado.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema consulta el calendario.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema detecta que todos los meses están pagados o que no existen periodos futuros.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema informa que no hay colegiaturas disponibles para adelantar.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El sistema no registra ningún pago.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El caso de uso termina. D — El número de meses supera los periodos disponibles<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario indica una cantidad mayor que los meses pendientes.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema detecta que no existen suficientes periodos.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema informa cuántos meses pueden pagarse.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema solicita corregir la cantidad.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El usuario selecciona un número válido.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El sistema vuelve a calcular el monto. E — El usuario cancela la operación<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema muestra el resumen del pago adelantado.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El usuario selecciona **Cancelar**.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema cierra el proceso.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. No se registra el pago.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. Los periodos conservan su estado anterior.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El calendario permanece sin cambios.  F — El monto cubre solo una parte de los meses seleccionados<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario ingresa un monto menor que el total.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema detecta que el importe no cubre todos los meses.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema informa que el pago adelantado debe cubrir completamente los periodos seleccionados.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema solicita reducir el número de meses o completar el monto.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El sistema no marca como pagados los periodos mientras el monto no sea suficiente. |
| **Precondiciones** | El usuario debe haber iniciado sesión. El alumno debe estar registrado. Debe existir un ciclo escolar activo. El alumno debe tener un plan y calendario de pagos. Deben existir meses futuros pendientes. Las tarifas de colegiatura deben estar configuradas. El usuario debe contar con permisos para registrar pagos. Los periodos que se pagarán no deben estar liquidados previamente. |
| **Postcondiciones** | El pago queda almacenado. Los meses cubiertos quedan en estado Pagado. El calendario de pagos queda actualizado. Los periodos cubiertos no generan recargos. Los periodos cubiertos no generan alertas de morosidad. La operación queda registrada en el historial. |
| **Reglas de negocio involucradas** | Solo pueden pagarse por adelantado colegiaturas futuras pendientes. Los periodos deben cubrirse en orden cronológico. El monto debe corresponder con el número de meses seleccionados. Los meses ya pagados no deben incluirse nuevamente. Un periodo pagado por adelantado no debe generar recargos. Un periodo pagado por adelantado no debe generar alertas de morosidad. El pago adelantado debe registrarse en el historial. El sistema debe mostrar claramente los meses que serán cubiertos. El pago debe cubrir completamente los periodos seleccionados. Los métodos permitidos son transferencia, depósito y tarjeta. |
