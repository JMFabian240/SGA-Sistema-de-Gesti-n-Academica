# RF-62: Adjuntar un comprobante digital de pago

| Campo | Descripción |
| :---: | ----- |
| **Autor** | Diana Jareth Ritt Martinez |
| **Nombre** | RF-62: Adjuntar un comprobante digital de pago |
| **Actor** | Administrador o Gestor Administrativo |
| **Objetivo** | Permitir al personal administrativo adjuntar una imagen o documento PDF como comprobante de un pago registrado, relacionándolo con el pago correspondiente y con el expediente del padre de familia o tutor para permitir su consulta y descarga posterior. |
| **Flujo Principal** | 1. El usuario accede al módulo de pagos.<br>2. Busca y selecciona al alumno.<br>3. El sistema muestra el historial de pagos.<br>4. El usuario selecciona el pago correspondiente.<br>5. Selecciona la opción **Adjuntar comprobante**.<br>6. Elige una imagen o archivo PDF.<br>7. El sistema valida el formato del archivo.<br>8. El sistema muestra la información del archivo y del pago.<br>9. El usuario confirma la operación.<br>10. El sistema almacena el comprobante y lo relaciona con el pago.<br>11. El sistema registra la operación.<br>12. El sistema muestra un mensaje de confirmación. |
| **Flujo Alterno** | Flujo alterno A — El archivo no tiene un formato permitido<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario selecciona un archivo.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema verifica el formato.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema detecta que el archivo no es una imagen ni un PDF.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema informa que el formato no está permitido.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El sistema no guarda el archivo.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El usuario selecciona otro archivo.<br>&nbsp;&nbsp;&nbsp;&nbsp;7. El flujo continúa desde la validación del formato. Flujo alterno B — El pago ya tiene un comprobante<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario selecciona un pago.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema detecta que ya existe un comprobante relacionado.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema muestra el comprobante actual.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El usuario puede consultarlo o descargarlo.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. Si cuenta con permisos, el usuario selecciona la opción para sustituirlo.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El sistema solicita confirmación.<br>&nbsp;&nbsp;&nbsp;&nbsp;7. El usuario confirma la sustitución.<br>&nbsp;&nbsp;&nbsp;&nbsp;8. El sistema guarda el nuevo archivo.<br>&nbsp;&nbsp;&nbsp;&nbsp;9. El sistema registra que el comprobante fue reemplazado. Flujo alterno C — El usuario cancela la carga<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario selecciona un archivo.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Antes de confirmar, selecciona **Cancelar**.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema cierra el proceso.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El archivo no queda almacenado.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El pago conserva la información anterior. Flujo alterno D — No existe un pago registrado<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario consulta el historial del alumno.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema detecta que no existen pagos registrados.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema informa que no es posible adjuntar un comprobante.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema no muestra la opción para guardar el archivo.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El caso de uso termina. Flujo alterno E — El usuario consulta un comprobante<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario selecciona un pago que tiene un comprobante.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra que existe un archivo relacionado.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El usuario selecciona **Consultar comprobante**.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema muestra el archivo.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El usuario puede revisar la información sin modificarla. Flujo alterno F — El usuario descarga un comprobante<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario selecciona un pago con comprobante.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El usuario selecciona la opción **Descargar**.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema prepara el archivo.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema proporciona el comprobante para su descarga.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El archivo original permanece relacionado con el pago. |
| **Precondiciones** | El usuario debe haber iniciado sesión. El usuario debe tener rol de Administrador o Gestor Administrativo. El alumno y el tutor deben estar registrados. Debe existir un pago registrado. El usuario debe tener acceso al comprobante digital. El archivo debe tener un formato permitido. El usuario debe contar con permisos para adjuntar documentos. |
| **Postcondiciones** | El archivo queda almacenado. El archivo queda relacionado con el pago. El comprobante queda disponible desde el expediente financiero del tutor. El usuario puede consultarlo o descargarlo posteriormente. La carga queda registrada. |
| **Reglas de negocio involucradas** | Solo se permiten imágenes y documentos PDF. El comprobante debe relacionarse con un pago existente. El archivo debe quedar relacionado con el expediente financiero del tutor. El comprobante debe poder consultarse posteriormente. El comprobante debe poder descargarse. El archivo no modifica el monto ni el estado del pago. La sustitución de un comprobante debe requerir autorización. Toda carga o sustitución debe identificar al usuario responsable. No debe guardarse un archivo cuando la carga no se haya completado correctamente. |
