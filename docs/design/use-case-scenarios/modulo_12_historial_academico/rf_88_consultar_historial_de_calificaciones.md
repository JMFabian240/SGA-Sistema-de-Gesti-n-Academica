# RF-88: Consultar historial de calificaciones

| Campo | Descripción |
| :---: | :---- |
| **Autor** | Jessica Pablo Hernández |
| **Nombre** | RF-88: Consultar historial de calificaciones |
| **Actor** | Administrador, gestor administrativo o docente. |
| **Objetivo** | Permitir la consulta organizada del historial de calificaciones de un alumno, mostrando sus registros académicos por ciclo escolar, periodo y materia, incluyendo calificaciones de clubes extracurriculares, para apoyar el seguimiento académico del estudiante. |
| **Flujo Principal** | **1\. Acceder al módulo de historial académico**  El actor autenticado selecciona el módulo **Historial Académico** desde el menú principal. El sistema valida que el usuario tenga permisos para consultar información académica. **2\. Seleccionar opción de historial de calificaciones**  El actor elige la opción **Consultar historial de calificaciones**. El sistema muestra la pantalla de búsqueda de alumnos. **3\. Buscar y seleccionar expediente del alumno**  El actor busca al alumno por nombre o matrícula. El sistema consulta la base de datos, muestra los expedientes coincidentes y permite seleccionar el expediente correspondiente. **4\. Validar expediente académico**  El sistema verifica que el alumno exista, que el expediente académico esté disponible y que cuente con información académica asociada. **5\. Recuperar registros de calificaciones**  El sistema consulta en la base de datos todas las calificaciones registradas del alumno, incluyendo materias curriculares y clubes extracurriculares. **6\. Clasificar registros académicos**  El sistema separa las calificaciones por tipo de registro académico, diferenciando materias curriculares, clubes extracurriculares y evaluaciones cualitativas cuando existan. **7\. Organizar información por ciclo escolar, por periodo y materia**  El sistema agrupa las calificaciones recuperadas según el ciclo escolar al que pertenecen. Dentro de cada ciclo escolar, el sistema organiza los registros por periodo académico y por materia o club extracurricular correspondiente. **8\. Validar integridad de información mostrada**  El sistema verifica que los registros recuperados correspondan al alumno seleccionado y que no existan datos duplicados o inconsistentes visibles para el usuario. **9\. Mostrar historial de calificaciones**  El sistema presenta en pantalla el historial académico organizado, mostrando ciclo escolar, periodo, materia, calificación o evaluación registrada y clubes extracurriculares cuando existan. **10\. Permitir navegación dentro del historial**  El actor puede expandir o contraer ciclos escolares, revisar periodos específicos o volver al listado de búsqueda sin alterar la información. **11\. Registrar consulta en bitácora, si aplica**  Si la política institucional lo requiere, el sistema registra la consulta realizada, incluyendo usuario, fecha, hora y expediente consultado. **12\. Finalizar consulta**  El caso de uso termina cuando el actor cierra la vista del historial, regresa al menú principal o busca otro alumno. |

El actor puede modificar el criterio de búsqueda o cancelar la consulta.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. Si modifica el criterio, el flujo regresa al paso<br>&nbsp;&nbsp;&nbsp;&nbsp;3. <br>&nbsp;&nbsp;&nbsp;&nbsp;5. Si cancela, el caso de uso termina sin mostrar información.  **A2. El alumno no tiene calificaciones registradas Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;5. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema no encuentra calificaciones asociadas al alumno.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra el mensaje: “El alumno no cuenta con calificaciones registradas”.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor puede regresar a la búsqueda de alumnos.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El caso de uso termina sin mostrar registros académicos. **A3. No existen clubes extracurriculares registrados Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;6. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema no encuentra calificaciones de clubes extracurriculares asociadas al alumno.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema omite la sección de clubes extracurriculares o la muestra como “Sin registros”.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema continúa mostrando las materias curriculares disponibles.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El flujo continúa desde el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;7. **A4. Existen evaluaciones cualitativas Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;6. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema detecta evaluaciones cualitativas, como Taller o evaluaciones descriptivas.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema las clasifica separadamente de las calificaciones numéricas.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema las muestra respetando su formato original.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El flujo continúa desde el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;7. **A5. El actor busca otro alumno Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;10. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El actor selecciona la opción de nueva búsqueda.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema limpia la pantalla del historial consultado.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. | **Precondiciones** | El actor debe haber iniciado sesión en el sistema. El actor debe contar con permisos para consultar información académica. El alumno debe estar registrado en el sistema. El expediente académico del alumno debe estar disponible. Deben existir registros académicos asociados al alumno o, en su defecto, el sistema debe poder indicar que aún no hay historial disponible. La base de datos académica debe estar accesible desde el cliente conectado al servidor local. |
| **Postcondiciones** | El historial de calificaciones del alumno se visualiza en pantalla. La información se muestra organizada por ciclo escolar, periodo y materia. Se incluyen calificaciones de clubes extracurriculares cuando existan. No se modifica ningún registro académico. La consulta puede quedar registrada en bitácora si la política institucional lo requiere. |
| **Reglas de negocio involucradas** | El historial debe organizarse por ciclo escolar, periodo y materia. Las calificaciones de clubes extracurriculares deben incluirse cuando existan. La consulta no debe modificar datos académicos. Solo usuarios autorizados pueden consultar historiales académicos. Los registros deben mostrarse asociados únicamente al alumno seleccionado. Las evaluaciones cualitativas deben mostrarse respetando su formato original. La información incompleta no debe inventarse ni calcularse sin registros existentes. Las consultas pueden registrarse en bitácora según políticas institucionales. |
