# RF-55: Configurar los montos para un nuevo ciclo escolar

| Campo | Descripción |
| :---: | ----- |
| **Autor** | Diana Jareth Ritt Martinez |
| **Nombre** | RF-55: Configurar los montos para un nuevo ciclo escolar |
| **Actor** | Administrador y Gestor Administrativo |
| **Objetivo** | Permitir al personal administrativo configurar los montos de colegiatura, inscripción, aranceles y materiales para un ciclo escolar determinado, diferenciándolos por nivel educativo y conservando sin cambios la información de ciclos anteriores. |
| **Flujo Principal** | 1. El usuario ingresa al módulo de ciclo escolar.<br>2. El sistema muestra los ciclos escolares disponibles.<br>3. El usuario selecciona el ciclo escolar que desea configurar.<br>4. El sistema muestra los niveles educativos registrados.<br>5. El usuario selecciona un nivel educativo.<br>6. El sistema muestra los conceptos configurables: colegiatura, inscripción, aranceles y materiales.<br>7. El usuario captura los montos correspondientes para cada concepto.<br>8. El sistema valida que los montos ingresados sean correctos.<br>9. El sistema muestra un resumen con el ciclo escolar, nivel educativo y montos registrados.<br>10. El usuario revisa la información capturada.<br>11. El usuario selecciona la opción Guardar.<br>12. El sistema solicita confirmación para registrar los cambios.<br>13. El usuario confirma la operación.<br>14. El sistema guarda la configuración de montos para el ciclo escolar seleccionado.<br>15. El sistema muestra un mensaje de confirmación. |
| **Flujo Alterno** | **A — Existen datos incompletos**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario intenta guardar la configuración.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema detecta que uno o más conceptos obligatorios no tienen monto.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema señala los datos faltantes.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema no guarda la configuración.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El usuario completa la información.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El flujo continúa desde el paso 9 del flujo principal. **B — Se ingresó un monto inválido**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario ingresa un monto negativo, igual a cero o con un formato no permitido.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema informa que el monto no es válido.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema solicita corregir el dato.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El usuario ingresa un nuevo monto.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El sistema vuelve a validar la información. **C — Ya existen tarifas para el ciclo y nivel**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario selecciona un ciclo y nivel que ya tienen montos configurados.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra los montos actuales.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El usuario modifica uno o más valores.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema muestra un resumen de los cambios.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El sistema solicita confirmación.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El usuario confirma.<br>&nbsp;&nbsp;&nbsp;&nbsp;7. El sistema actualiza los montos del ciclo indicado.<br>&nbsp;&nbsp;&nbsp;&nbsp;8. Los registros históricos de ciclos anteriores permanecen sin cambios. **D — El ciclo escolar está cerrado**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario selecciona un ciclo escolar.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema detecta que el ciclo está cerrado.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema informa que no se pueden modificar sus tarifas.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema permite únicamente consultar los montos registrados.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El caso de uso termina sin realizar cambios. **E — El usuario cancela la operación**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El usuario ingresa o modifica los montos.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Antes de confirmar, selecciona **Cancelar**.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema descarta los cambios no guardados.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. Los montos anteriores permanecen sin modificaciones.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El sistema regresa a la consulta de configuración financiera. |
| **Precondiciones** | 1\. 	El usuario debe haber iniciado sesión. 2\. 	El usuario debe tener rol de Administrador o Gestor Administrativo. 3\. 	El ciclo escolar que será configurado debe estar registrado. 4\. 	Los niveles educativos deben estar registrados en el sistema. 5\. 	El ciclo escolar no debe estar cerrado. 6\. 	El usuario debe contar con permisos para configurar montos financieros. |
| **Postcondiciones** | **Si la configuración se guarda correctamente:** 1\. 	Los montos quedan relacionados con el ciclo escolar y nivel educativo indicados. 2\. 	Los montos quedan disponibles para la generación de cobros del nuevo ciclo. 3\. 	Los ciclos escolares anteriores permanecen sin cambios. 4\. 	La configuración queda registrada en la bitácora. **Si la operación no se completa:** 1\. 	Los montos anteriores permanecen sin cambios. 2\. 	No se crean registros incompletos. 3\. 	El usuario recibe información sobre el error. |
| **Reglas de negocio involucradas** | • Las tarifas deben definirse por ciclo escolar.<br>• Las tarifas deben diferenciarse por nivel educativo.<br>• Los conceptos configurables incluyen colegiatura, inscripción, aranceles y materiales.<br>• Los nuevos montos solo deben aplicarse al ciclo indicado.<br>• Los cambios no deben alterar los históricos de ciclos anteriores.<br>• El sistema debe solicitar confirmación antes de guardar.<br>• Los montos deben ser mayores que cero.<br>• Un ciclo escolar cerrado no puede modificarse.<br>• Toda configuración o modificación debe registrarse en la bitácora. |
