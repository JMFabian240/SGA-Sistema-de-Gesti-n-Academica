# RF-30: Filtrar listado de alumnos

| Campo | Descripción |
| :---: | ----- |
| **Autor** | José Manuel Fabian Hernández |
| **Nombre** | RF-30: Filtrar listado de alumnos |
| **Actor** | Administrador y Gestor Administrativo |
| **Objetivo** | Permitir al personal administrativo aplicar filtros combinados sobre el directorio de alumnos (nivel educativo, grado y grupo) para obtener un subconjunto específico del listado, con opción de exportar el resultado. |
| **Flujo Principal** | 1. El actor (Administrador o Gestor) ingresa al módulo de Gestión de Alumnos.<br>2. El sistema despliega el listado completo de alumnos registrados con sus datos básicos.<br>3. El actor selecciona uno o más criterios de filtrado disponibles: nivel educativo, grado o grupo. Adicionalmente, el actor puede activar la opción "Incluir alumnos inactivos o dados de baja" si desea ampliar el alcance de la búsqueda.<br>5. El sistema procesa los criterios seleccionados y despliega únicamente los alumnos que coincidan con todos los filtros aplicados.<br>6. El actor visualiza el listado filtrado con el nombre del alumno, grado, grupo, nivel y estado del expediente.<br>7. Opcionalmente, el actor hace clic en "Exportar" y selecciona el formato CSV.<br>8. El sistema genera y descarga el archivo con el listado filtrado en el formato elegido. |
| **Flujo Alterno** | <b>A.</b> Ningún alumno coincide con los filtros:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 5, el sistema no encuentra alumnos que cumplan con todos los criterios seleccionados.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema despliega el mensaje: "No se encontraron alumnos con los criterios de filtrado aplicados".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor modifica o elimina alguno de los filtros y vuelve a intentar.<br><br><b>B.</b> Error al generar la exportación:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 8, el sistema experimenta un error al intentar generar el archivo.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra el mensaje: "No se pudo generar el archivo. Intente nuevamente".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor reintenta la exportación. |
| **Precondiciones** | El actor debe tener sesión activa con los privilegios de Administrador o Gestor Administrativo. Deben existir alumnos registrados en el sistema. |
| **Postcondiciones** | El actor obtiene el subconjunto de alumnos que cumplen los criterios seleccionados. Si se solicitó exportación, el archivo queda disponible para descarga inmediata. No se realiza ninguna modificación en la base de datos. |
| **Reglas de negocio involucradas** | • Los filtros pueden aplicarse de forma individual o combinada (nivel, grado, grupo) para mayor precisión en la consulta.<br>• Por defecto, el listado filtrado muestra únicamente alumnos con expediente activo. El actor puede activar expresamente la opción de incluir alumnos inactivos o dados de baja para ampliar el resultado.<br>• El archivo exportado debe contener exactamente los mismos registros y datos visibles en pantalla con los filtros aplicados. |
