# RF-29: Modificar datos de alumno

| Campo | Descripción |
| :---: | ----- |
| **Autor** | José Manuel Fabian Hernández |
| **Nombre** | RF-29: Modificar datos de alumno |
| **Actor** | Administrador y Gestor Administrativo |
| **Objetivo** | Actualizar o corregir la información del expediente de un estudiante (grado, grupo, personas autorizadas para recogerlo u otros datos escolares), asegurando que el sistema cuente con datos veraces y vigentes. |
| **Flujo Principal** | 1. El actor (Administrador o Gestor) ingresa al módulo de Gestión de Alumnos.<br>2. Utiliza la barra de búsqueda para localizar al alumno por nombre o matrícula.<br>3. El actor selecciona el expediente deseado de la lista de resultados.<br>4. El actor hace clic en la opción "Editar" o "Modificar datos".<br>5. El sistema habilita los campos del formulario con la información actual precargada.<br>6. El actor ingresa la información corregida o actualizada (grado, grupo, personas autorizadas u otros datos escolares permitidos).<br>7. El actor hace clic en "Guardar cambios".<br>8. El sistema valida que los nuevos datos sean correctos, que no falte información obligatoria y que los valores de CURP y matrícula, en caso de ser modificados, no estén ya asignados a otro alumno en el sistema.<br>9. El sistema actualiza la base de datos de forma inmediata, registra la acción en la bitácora de auditoría y muestra el mensaje: "Datos actualizados correctamente". |
| **Flujo Alterno** | <b>A.</b> Cancelación de la edición:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 6, el actor decide no aplicar ninguna actualización y hace clic en "Cancelar".<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema descarta los cambios en pantalla y devuelve al actor a la vista detallada del expediente original sin alterar la base de datos.<br><br><b>B.</b> Dato obligatorio eliminado o inválido:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 8, el sistema detecta que el actor borró un campo obligatorio (como la CURP o matrícula) o ingresó un formato inválido.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema detiene la actualización, resalta el campo erróneo en rojo y muestra el mensaje: "Por favor, complete correctamente los campos obligatorios".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor corrige el dato y vuelve a intentar guardar.<br><br><b>C.</b> CURP o matrícula duplicada:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 8, el sistema detecta que el valor de CURP o matrícula ingresado ya pertenece a otro expediente registrado en el sistema.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema detiene la actualización, resalta el campo en conflicto en rojo y muestra el mensaje: "La CURP o matrícula ingresada ya está asignada a otro alumno. Verifique la información".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor corrige el dato y vuelve a intentar guardar. |
| **Precondiciones** | El actor debe tener sesión activa con los permisos de Administrador o Gestor Administrativo. El expediente del alumno debe existir previamente en el sistema. |
| **Postcondiciones** | La base de datos queda actualizada con la nueva información. Los datos modificados (como el cambio de grado o grupo del alumno) se reflejan inmediatamente en los demás módulos del sistema. La acción queda registrada en la bitácora de auditoría (RF-08). |
| **Reglas de negocio involucradas** | • La modificación de expedientes de alumnos está habilitada para los perfiles Administrador y Gestor Administrativo.<br>• Los campos de CURP y matrícula son únicos por expediente; el sistema debe impedir la asignación de valores que ya pertenezcan a otro alumno.<br>• Cualquier modificación realizada queda registrada automáticamente en la bitácora de auditoría del sistema (RF-08). |
