# RF-31: Desactivar expediente de alumno

| Campo | Descripción |
| :---: | ----- |
| **Autor** | José Manuel Fabian Hernández |
| **Nombre** | RF-31: Desactivar expediente de alumno |
| **Actor** | Administrador y Gestor Administrativo |
| **Objetivo** | Registrar la salida de un alumno que egresó o se dio de baja del plantel, ocultando su expediente de las vistas activas del sistema sin eliminarlo permanentemente, y conservando su historial académico y financiero para auditorías futuras. |
| **Flujo Principal** | 1. El actor (Administrador o Gestor) ingresa al módulo de Gestión de Alumnos.<br>2. Utiliza la barra de búsqueda para localizar al alumno por nombre o matrícula.<br>3. El sistema muestra el expediente correspondiente en la lista de resultados.<br>4. El actor selecciona el expediente y elige la opción "Desactivar expediente" o "Dar de baja".<br>5. El sistema despliega un cuadro de confirmación: "¿Está seguro de que desea desactivar el expediente de este alumno• El historial académico y financiero se conservará para consultas futuras".<br>6. El actor hace clic en "Confirmar".<br>7. El sistema cambia el estado del expediente a "Inactivo" y lo excluye de los listados activos de alumnos.<br>8. El sistema registra la acción en la bitácora de auditoría y muestra el mensaje de éxito: "Expediente desactivado correctamente". |
| **Flujo Alterno** | <b>A.</b> Cancelación de la desactivación:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 5, el actor decide no proceder (por ejemplo, seleccionó al alumno equivocado).<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El actor hace clic en "Cancelar".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema cierra el cuadro de confirmación sin realizar ningún cambio y regresa al listado general.<br><br><b>B.</b> Alumno con adeudos pendientes:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 6, el sistema detecta que el alumno tiene conceptos de pago pendientes o vencidos.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra la advertencia: "El alumno tiene adeudos registrados. Se recomienda regularizar su situación financiera antes de desactivar el expediente".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor puede continuar con la desactivación o cancelar para atender primero la situación financiera. |
| **Precondiciones** | El actor debe tener sesión activa con los privilegios correspondientes. El expediente del alumno debe existir y estar activo en el sistema. |
| **Postcondiciones** | El expediente queda desactivado y excluido de los listados activos. El historial académico y financiero se conserva íntegramente para consultas y auditorías futuras (RNF-09). La acción queda registrada en la bitácora de auditoría (RF-08). |
| **Reglas de negocio involucradas** | • La desactivación de expedientes está permitida para los perfiles Administrador y Gestor Administrativo.<br>• La desactivación no elimina permanentemente el expediente; el historial académico y financiero se conserva siempre para auditorías futuras (RNF-09).<br>• Un expediente desactivado puede consultarse, pero no puede recibir nuevos registros de pagos, calificaciones ni cambios hasta ser reactivado. |
