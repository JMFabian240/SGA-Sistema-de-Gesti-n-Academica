# RF-39: Ejecutar baja definitiva de alumno

| Campo | Descripción |
| ----- | ----- |
| **Autor** | José Manuel Fabian Hernández |
| **Nombre** | RF-39: Ejecutar baja definitiva de alumno |
| **Actor** | Administrador |
| **Objetivo** | Formalizar la salida permanente de un alumno con estatus de "Baja Temporal" que no haya regularizado su situación de pago, cerrando su expediente para el ciclo vigente y registrando la acción en la bitácora de auditoría. |
| **Flujo Principal** | 1. El Administrador ingresa al módulo de Gestión de Alumnos.<br>2. Aplica un filtro para visualizar únicamente los alumnos con estatus "Baja Temporal".<br>3. El sistema muestra el listado de alumnos con dicho estatus.<br>4. El Administrador selecciona el expediente del alumno al que desea formalizar la baja definitiva.<br>5. El sistema despliega el cuadro de confirmación: "¿Está seguro de ejecutar la baja definitiva de este alumno• Esta acción cerrará su expediente para el ciclo vigente".<br>6. El Administrador hace clic en "Confirmar baja definitiva".<br>7. El sistema cambia el estatus del alumno a "Baja Definitiva", cierra el expediente activo para el ciclo en curso y detiene la generación automática de nuevas colegiaturas.<br>8. El sistema registra la acción en la bitácora de auditoría y muestra el mensaje: "Baja definitiva ejecutada correctamente". |
| **Flujo Alterno** | <b>A.</b> Alumno sin estatus de Baja Temporal:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 4, el Administrador intenta ejecutar la baja definitiva de un alumno que no tiene el estatus "Baja Temporal".<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema bloquea la acción y muestra el mensaje: "Solo es posible ejecutar la baja definitiva de alumnos con estatus de Baja Temporal".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El Administrador regresa al listado y selecciona el expediente correcto.<br><br><b>B.</b> Cancelación de la baja definitiva:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 5, el Administrador decide no proceder.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El Administrador hace clic en "Cancelar".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema cierra el cuadro de confirmación sin realizar ningún cambio y regresa al listado general. |
| **Precondiciones** | El Administrador debe tener sesión activa. El alumno debe estar previamente en estatus "Baja Temporal" y no haber concretado un acuerdo de pago que justifique su reactivación. |
| **Postcondiciones** | El expediente del alumno queda cerrado para el ciclo vigente con estatus "Baja Definitiva". El historial académico y financiero se conserva para auditorías futuras (RNF-09). La generación automática de nuevas colegiaturas queda detenida. La acción queda registrada en la bitácora de auditoría (RF-08). |
| **Reglas de negocio involucradas** | • La ejecución de la baja definitiva es una función exclusiva del actor Administrador.<br>• Solo pueden recibir baja definitiva los alumnos que se encuentren previamente en estatus "Baja Temporal".<br>• La baja definitiva no elimina el historial del alumno; los datos académicos y financieros se conservan permanentemente para auditorías (RNF-09).<br>• La acción debe registrarse de forma obligatoria en la bitácora de auditoría (RF-08). |
