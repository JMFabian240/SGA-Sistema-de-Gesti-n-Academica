# RF-79: Registrar calificaciones de primaria

| Campo | Descripción |
| :---- | :---- |
| **Autor** | Jessica Pablo Hernández |
| **Nombre** | RF-79: Registrar calificaciones de primaria |
| **Actor** | Administrador, gestor administrativo o docente |
| **Objetivo** | Permitir el registro formal de las calificaciones numéricas de alumnos de primaria por materia y trimestre, asegurando que el sistema almacene la información académica en el ciclo escolar activo y calcule automáticamente el promedio final cuando corresponda. |
| **Flujo Principal** | **1\. Acceder al módulo de calificaciones**  El actor autenticado selecciona el módulo **Registro de Calificaciones**. El sistema valida sus permisos según el rol. **2\. Seleccionar nivel educativo primaria:** El actor selecciona el nivel educativo Primaria. El sistema configura la búsqueda y la estructura de captura correspondiente a calificaciones trimestrales de primaria. **3\. Buscar y seleccionar expediente del alumno**  El actor busca al alumno por nombre o matrícula. El sistema consulta únicamente alumnos de primaria vinculados al ciclo escolar activo y muestra los expedientes coincidentes. **4\. Validar expediente académico**  El sistema verifica que el expediente seleccionado exista, esté vinculado al ciclo escolar activo y cuente con grado, grupo y materias asociadas. **5\. Cargar materias y periodos disponibles**  El sistema consulta las materias asociadas al grado y grupo del alumno y muestra la estructura de captura por materia y trimestre. **6\. Seleccionar trimestre de captura**  El actor selecciona el trimestre que desea registrar. El sistema valida que el trimestre corresponda al ciclo escolar activo. **7\. Capturar calificaciones numéricas**  El actor ingresa la calificación numérica correspondiente a cada materia del trimestre seleccionado.  **8\. Validar calificaciones capturadas**  El sistema valida que las calificaciones tengan formato numérico, que no existan campos obligatorios vacíos y que no se dupliquen registros ya existentes. **9\. Confirmar guardado**  El actor selecciona Guardar calificaciones y confirma la operación. El sistema almacena cada calificación vinculándola con alumno, materia, trimestre, ciclo escolar activo y usuario responsable. **10\. Calcular promedio final cuando aplique**  El sistema verifica si existen calificaciones de los tres trimestres requeridos y, de ser así, calcula automáticamente el promedio final del ciclo escolar. **11\. Actualizar expediente y boleta**  El sistema actualiza el expediente académico y deja las calificaciones disponibles para consulta, historial académico y generación de boleta. **12\. Registrar auditoría**  El sistema registra en bitácora la operación realizada, incluyendo usuario, fecha, hora, alumno, trimestre y acción ejecutada. **13\. Mostrar confirmación final**  El sistema informa que las calificaciones de primaria fueron registradas correctamente. |
| **Flujo Alterno** | **A1. Alumno no encontrado Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;3. **  El sistema no encuentra coincidencias. Muestra mensaje informativo y permite al actor ingresar otro criterio de búsqueda. **A2. Calificaciones ya registradas Ocurre en el paso 8**  El sistema detecta registros existentes para el alumno, materia y trimestre. Muestra la información en modo consulta e indica que los cambios deben realizarse mediante el caso de uso **Modificar calificación registrada**. **A3. Captura parcial de materias Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;8. **  El sistema detecta materias sin calificación. Muestra advertencia indicando que el trimestre quedará incompleto. Si el actor confirma, guarda solo las materias capturadas sin calcular promedio final, si el actor no confirma, el sistema conserva los datos en pantalla para edición. **A4. El actor cancela el registro Ocurre entre los pasos 7 y<br>&nbsp;&nbsp;&nbsp;&nbsp;9. **  El actor selecciona **Cancelar**. El sistema solicita confirmación y, si se acepta, descarta los datos temporales sin guardar cambios. **A5. El actor no confirma el guardado Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;9. **  El actor rechaza la confirmación. El sistema conserva los datos en pantalla para edición o cancelación, sin almacenarlos y regresa al paso<br>&nbsp;&nbsp;&nbsp;&nbsp;7.  |
| **Precondiciones** | El actor debe haber iniciado sesión en el sistema. El actor debe contar con permisos para acceder al módulo de registro de calificaciones. El alumno debe pertenecer al nivel educativo primaria. Debe existir un ciclo escolar activo. El expediente académico del alumno debe estar disponible. Las materias correspondientes al alumno deben estar registradas para el ciclo escolar activo. |
| **Postcondiciones** | Las calificaciones numéricas quedan registradas por alumno, materia, trimestre y ciclo escolar activo. Si se registran las calificaciones de los tres trimestres, el sistema genera automáticamente el promedio final del ciclo. Las calificaciones quedan disponibles para consulta, historial académico y generación de boleta. La operación queda registrada en bitácora de auditoría. |
| **Reglas de negocio involucradas** | ·         Las calificaciones de primaria deben ser numéricas. ·         Las calificaciones deben asociarse a una materia, trimestre, alumno y ciclo escolar activo. ·         El sistema debe generar automáticamente el promedio final cuando existan calificaciones de los tres trimestres. ·         Una calificación ya registrada no debe sobrescribirse directamente. ·         Las modificaciones posteriores deben realizarse mediante el caso de uso correspondiente y con motivo de modificación. ·         Las calificaciones registradas deben quedar disponibles para boleta e historial académico. ·         El sistema debe mantener trazabilidad de las operaciones académicas relevantes. |
