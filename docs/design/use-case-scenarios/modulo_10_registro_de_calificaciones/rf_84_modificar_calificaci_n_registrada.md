# RF-84: Modificar calificación registrada

| Campo | Descripción |
| :---- | :---- |
| **Autor** | Jessica Pablo Hernández |
| **Nombre** | RF-84: Modificar calificación registrada |
| **Actor** | Administrador, gestor administrativo o docente |
| **Objetivo** | Permitir la modificación controlada de calificaciones previamente registradas, garantizando la actualización correcta de la información académica y conservando trazabilidad completa de los cambios realizados. |
| **Flujo Principal** | **1\. Acceder al módulo de calificaciones**  El actor autenticado ingresa al módulo **Registro de Calificaciones**. El sistema valida que el usuario tenga permisos para consultar y modificar información académica. **2\. Buscar y seleccionar expediente del alumno**  El actor busca al alumno por nombre o matrícula. El sistema consulta la base de datos, muestra los expedientes coincidentes y permite seleccionar el expediente correspondiente. **3\. Validar expediente académico**  El sistema verifica que el alumno exista y que el expediente académico esté disponible y asociado al ciclo escolar correspondiente. **4\. Consultar calificaciones registradas**  El sistema recupera las calificaciones registradas del alumno, incluyendo materias curriculares, talleres, clubes extracurriculares o evaluaciones cualitativas disponibles para modificación. **5\. Seleccionar calificación a modificar**  El actor selecciona la calificación específica que desea modificar. El sistema muestra la información actual registrada, incluyendo materia, periodo, valor registrado y fecha de captura original. **6\. Capturar nueva calificación o evaluación**  El actor ingresa el nuevo valor de la calificación o evaluación correspondiente. El sistema mantiene temporalmente el nuevo dato sin reemplazar aún el registro original. **7\. Capturar motivo de modificación**  El actor registra el motivo obligatorio de la modificación. El sistema valida que el motivo no se encuentre vacío. **8\. Validar información modificada**  El sistema valida el formato de la nueva calificación según el tipo de evaluación: numérica para materias curriculares o clubes; A o NA para Taller; texto descriptivo para evaluaciones de preescolar. El sistema también valida que el motivo de modificación no esté vacío. **9\. Confirmar modificación**  El actor selecciona **Guardar modificación**. El sistema muestra una confirmación indicando que la calificación original será actualizada y que la operación quedará registrada en auditoría. **10\. Actualizar calificación registrada**  El actor confirma la operación. El sistema actualiza la calificación en la base de datos, conservando referencia del valor anterior y registrando usuario, fecha, hora y motivo de modificación. **11\. Actualizar historial académico y boletas**  El sistema actualiza el historial académico, boletas y, si la calificación afecta promedios, los recalcula automáticamente. **12\. Registrar auditoría completa**  El sistema registra en bitácora la modificación realizada, incluyendo valor anterior, valor nuevo, usuario responsable, fecha, hora y motivo del cambio. **13\. Mostrar confirmación final**  El sistema informa que la calificación fue modificada correctamente. |

| **Precondiciones** | El actor debe haber iniciado sesión en el sistema. El actor debe contar con permisos para modificar calificaciones. Debe existir un ciclo escolar activo o histórico consultable. El alumno debe estar registrado en el sistema. Debe existir una calificación previamente registrada. El expediente académico del alumno debe estar disponible. |
| **Postcondiciones** | La calificación seleccionada queda modificada correctamente. El historial académico del alumno queda actualizado. El sistema conserva la calificación anterior en registros de auditoría. La modificación queda registrada con usuario, fecha, hora y motivo del cambio. Si aplica, el sistema recalcula promedios académicos relacionados. La información actualizada queda disponible para consulta y generación de boletas. |
| **Reglas de negocio involucradas** | Toda modificación requiere motivo. Toda modificación debe registrarse en bitácora. Si la calificación afecta promedio, el sistema debe recalcularlo. No se debe perder el valor anterior. |
