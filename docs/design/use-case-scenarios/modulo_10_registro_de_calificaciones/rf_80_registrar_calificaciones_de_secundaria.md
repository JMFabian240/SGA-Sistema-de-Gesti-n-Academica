# RF-80: Registrar calificaciones de secundaria

| Campo | Descripción |
| :---- | :---- |
| **Autor** | Jessica Pablo Hernández |
| **Nombre** | RF-80: Registrar calificaciones de secundaria |
| **Actor** | Administrador, gestor administrativo o docente |
| **Objetivo** | Permitir el registro formal de calificaciones de secundaria por materia y bloque, asegurando que la información académica quede vinculada al expediente del alumno y al ciclo escolar activo. |
| **Flujo Principal** | **1\. Acceder al módulo de calificaciones**  El actor autenticado ingresa al módulo **Registro de Calificaciones**. El sistema valida que su rol tenga permisos para registrar calificaciones. **2\. Seleccionar nivel educativo secundaria** El actor selecciona el nivel educativo **Secundaria**. El sistema configura la búsqueda y la estructura de captura correspondiente a **Bloque 1, Bloque 2, Bloque 3 y calificación final**. **3\. Buscar y seleccionar expediente del alumno**  El actor busca al alumno por nombre o matrícula. El sistema consulta la base de datos, muestra los resultados coincidentes y permite seleccionar el expediente.  **4\. Validar expediente académico**  El sistema verifica que el expediente seleccionado exista, esté vinculado al ciclo escolar activo y cuente con grado, grupo y materias asociadas. **5\. Cargar materias y estructura de evaluación**  El sistema consulta las materias asociadas al grado y grupo del alumno y muestra la estructura de secundaria: **Bloque 1, Bloque 2, Bloque 3 y calificación final**. **6\. Seleccionar materia y bloque**  El actor selecciona la materia y el bloque que desea registrar. El sistema valida que la materia pertenezca al alumno y que el bloque sea válido para secundaria. El actor puede repetir este paso para capturar otra materia o bloque sin reiniciar el caso de uso. **7\. Verificar calificación existente**  El sistema consulta si ya existe una calificación registrada para el alumno, materia, bloque y ciclo escolar activo. **8\. Capturar calificación**  El actor ingresa la calificación numérica correspondiente al bloque seleccionado. **9\. Validar calificación capturada**  El sistema valida que la calificación tenga formato numérico y que no exista duplicidad para el mismo alumno, materia, bloque y ciclo. **10\. Confirmar registro**  El actor selecciona **Guardar calificación**. El sistema muestra una confirmación indicando que el dato será almacenado en el expediente académico. **11\. Guardar calificación**  El actor confirma la operación. El sistema almacena la calificación en la base de datos, vinculándola con alumno, materia, bloque, ciclo escolar activo y usuario responsable. Si el bloque seleccionado corresponde a calificación final, el sistema la registra como resultado final de la materia. **12\. Actualizar expediente académico**  El sistema actualiza el expediente académico y deja la información disponible para consulta, historial académico y generación de boleta. **13\. Registrar auditoría**  El sistema registra la acción en bitácora, incluyendo usuario, fecha, hora, expediente afectado, materia, bloque y tipo de operación. **14\. Mostrar confirmación final**  El sistema informa que la calificación de secundaria fue registrada correctamente. |
| **Flujo Alterno** | **A1. Alumno no encontrado Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;3. **  El sistema no encuentra expedientes coincidentes. Muestra el mensaje correspondiente y permite al actor ingresar otro criterio de búsqueda. **A2. Ya existe calificación registrada Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;7. **  El sistema detecta que ya existe calificación para el alumno, materia, bloque y ciclo. Muestra el registro en modo consulta e indica que cualquier cambio debe realizarse mediante el caso de uso **Modificar calificación registrada**. **A3. Calificación inválida Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;9. **  El sistema detecta que el valor ingresado no es numérico o contiene caracteres no permitidos. Resalta el campo y solicita corregirlo. **A4. El actor cancela antes de guardar Ocurre entre los pasos 8 y<br>&nbsp;&nbsp;&nbsp;&nbsp;10. **  El actor selecciona **Cancelar**. El sistema solicita confirmación y descarta los datos temporales si el actor confirma. **A5. El actor no confirma el guardado Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;10. **  El actor rechaza la confirmación. El sistema no guarda información y permite continuar. |
| **Precondiciones** | 1\.      El actor debe haber iniciado sesión. 2\.      El actor debe contar con permisos para registrar calificaciones. 3\.      El alumno debe estar registrado. 4\.      El alumno debe pertenecer al nivel educativo secundaria. 5\.      Debe existir un ciclo escolar activo. 6\.      El alumno debe tener materias asociadas a su grado y grupo. 7\.      El expediente académico debe estar disponible en la base de datos |
| **Postcondiciones** | 1\.      Las calificaciones quedan registradas por alumno, materia, bloque y ciclo escolar activo. 2\.      La calificación final queda almacenada cuando sea capturada. 3\.      La información queda disponible para consulta, historial académico y generación de boleta. 4\.      La operación queda registrada en bitácora de auditoría cuando se crea o actualiza información académica. |
| **Reglas de negocio involucradas** | Solo alumnos de secundaria pueden ser evaluados mediante este caso de uso. Las calificaciones de secundaria se registran por materia y bloque. Los bloques válidos son: Bloque 1, Bloque 2, Bloque 3 y calificación final. Cada calificación debe vincularse al perfil del alumno y al ciclo escolar activo. No debe duplicarse una calificación para el mismo alumno, materia, bloque y ciclo escolar. Las calificaciones ya registradas no deben sobrescribirse directamente. Las modificaciones deben realizarse mediante un proceso de modificación con motivo. Las calificaciones deben quedar disponibles para consulta, historial académico y boleta. |
