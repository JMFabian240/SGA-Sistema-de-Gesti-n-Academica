# RF-83: Registrar evaluación de Taller

| Campo | Descripción |
| :---- | :---- |
| **Autor** | Jessica Pablo Hernández |
| **Nombre** | RF-83: Registrar evaluación de Taller |
| **Actor** | Administrador, gestor administrativo o docente |
| **Objetivo** | Permitir el registro de la evaluación cualitativa de la materia Taller mediante la escala “A” o “NA”, asegurando que esta evaluación quede almacenada en el historial académico del alumno sin afectar el promedio numérico general. |
| **Flujo Principal** | **1\. Acceder al módulo de calificaciones**  El actor autenticado selecciona el módulo **Registro de Calificaciones** desde el menú principal. El sistema valida que el usuario tenga permisos para registrar evaluaciones académicas.  **2\. Seleccionar materia Taller**  El actor selecciona la opción **Materia Taller**. El sistema configura la captura para evaluación cualitativa con los valores permitidos **A** y **NA**. **3\. Buscar y seleccionar expediente del alumno**  El actor busca al alumno por nombre o matrícula. El sistema consulta alumnos registrados en el ciclo escolar activo y muestra los expedientes coincidentes. **4\. Validar expediente académico**  El sistema verifica que el expediente exista, esté asociado al ciclo escolar activo y que la materia Taller esté asignada al alumno seleccionado. **5\. Seleccionar periodo académico**  El actor selecciona el periodo académico correspondiente. El sistema valida que el periodo pertenezca al ciclo escolar activo. **6\. Verificar evaluación existente**  El sistema consulta si ya existe una evaluación registrada para la materia Taller en el periodo seleccionado. **7\. Registrar evaluación cualitativa**  El actor selecciona la evaluación correspondiente. El sistema mantiene temporalmente el dato hasta confirmar el guardado. **8\. Validar evaluación capturada**  El sistema valida que el valor seleccionado corresponda únicamente a “A” o “NA”, evitando el registro de valores numéricos u otros formatos no permitidos. **9\. Guardar evaluación de Taller**  El actor confirma la operación. El sistema registra la evaluación en la base de datos, vinculándola con alumno, materia Taller, periodo, ciclo escolar activo y usuario responsable, clasificándola automáticamente como cualitativa y excluyéndola del cálculo de promedio numérico. **10\. Actualizar historial académico**  El sistema actualiza el historial académico del alumno y deja disponible la evaluación para consulta y generación de boleta. **11\. Registrar auditoría**  El sistema registra la operación en bitácora, incluyendo usuario, fecha, hora, expediente afectado, periodo y tipo de evaluación registrada. **12\. Mostrar confirmación final**  El sistema informa que la evaluación de Taller fue registrada correctamente. |
| **Flujo Alterno** | **A1. Alumno no encontrado Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;3. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema no encuentra expedientes coincidentes.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra el mensaje: “No se encontraron alumnos con los datos ingresados”.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor puede modificar el criterio de búsqueda o cancelar el proceso.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. Si modifica el criterio, el flujo regresa al paso<br>&nbsp;&nbsp;&nbsp;&nbsp;3. <br>&nbsp;&nbsp;&nbsp;&nbsp;5. Si cancela, el caso de uso termina sin cambios.  **A2. Ya existe evaluación registrada Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;6. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema detecta que ya existe una evaluación registrada para Taller en el mismo periodo y ciclo escolar.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra la evaluación existente en modo consulta.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema informa que la evaluación no puede sobrescribirse directamente.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema indica que cualquier cambio debe realizarse mediante el caso de uso **Modificar calificación registrada**.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El caso de uso termina sin modificar información.  **A3. El actor cancela antes de guardar Ocurre entre los pasos 7 y<br>&nbsp;&nbsp;&nbsp;&nbsp;8. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El actor selecciona **Cancelar**.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema solicita confirmación indicando que los datos no guardados se perderán.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. Si el actor confirma, el sistema descarta la información temporal.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema regresa a la pantalla anterior.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El caso de uso termina sin cambios.  **A4. El actor no confirma el guardado Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;9. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema solicita confirmación para guardar la evaluación.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El actor selecciona **No** o **Cancelar**.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema conserva la información en pantalla sin almacenarla.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El actor puede continuar editando o cancelar el proceso.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El flujo regresa al paso<br>&nbsp;&nbsp;&nbsp;&nbsp;7.  |
| **Precondiciones** | El actor debe haber iniciado sesión en el sistema. El actor debe contar con permisos para registrar evaluaciones académicas. El alumno debe estar registrado en el sistema. El expediente académico del alumno debe estar disponible. Debe existir un ciclo escolar activo. La materia Taller debe estar asociada al expediente académico del alumno. Debe existir un periodo académico válido para registrar la evaluación. |
| **Postcondiciones** | La evaluación de Taller queda registrada en el historial académico del alumno. La evaluación queda vinculada al alumno, periodo y ciclo escolar activo. La evaluación queda clasificada como evaluación cualitativa. La evaluación no afecta el promedio numérico general del alumno. La operación queda registrada en bitácora de auditoría cuando se genera el registro académico. |
| **Reglas de negocio involucradas** | 1\.      La materia Taller utiliza evaluación cualitativa.  2\.      Los únicos valores válidos son:  a.      “A” \= Acreditado  b.     “NA” \= No Acreditado  3\.      La evaluación debe vincularse al alumno, periodo y ciclo escolar activo.  4\.      La evaluación de Taller no debe afectar el promedio numérico general.  5\.      No debe duplicarse una evaluación para el mismo alumno y periodo.  6\.      Una evaluación ya registrada no debe sobrescribirse directamente.  7\.      Las modificaciones deben realizarse mediante el caso de uso correspondiente y con motivo de modificación.  8\.      La evaluación debe quedar disponible para historial académico y boleta. |
