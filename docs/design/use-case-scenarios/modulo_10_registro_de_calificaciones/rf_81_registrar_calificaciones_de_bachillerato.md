# RF-81: Registrar calificaciones de bachillerato

| Campo | Descripción |
| :---- | :---- |
| **Autor** | Jessica Pablo Hernández |
| **Nombre** | RF-81: Registrar calificaciones de bachillerato |
| **Actor** | Administrador, gestor administrativo o docente |
| **Objetivo** | Permitir el registro formal de calificaciones de alumnos de bachillerato por materia y bimestre, asegurando que la información académica quede asociada al expediente del alumno, al semestre en curso y al ciclo escolar activo. |
| **Flujo Principal** | **1\. Acceder al módulo de calificaciones**  El actor autenticado selecciona el módulo **Registro de Calificaciones** desde el menú principal. El sistema valida que su rol tenga permisos para registrar calificaciones académicas. **2\. Seleccionar nivel educativo bachillerato**  El actor selecciona el nivel educativo **Bachillerato**. El sistema configura la búsqueda y la estructura de captura correspondiente a materias por bimestre del semestre en curso. **3\. Buscar y seleccionar expediente del alumno**  El actor busca al alumno por nombre o matrícula. El sistema consulta únicamente alumnos de bachillerato vinculados al ciclo escolar activo y muestra los expedientes coincidentes. **4\. Validar expediente académico**  El sistema verifica que el expediente seleccionado exista, esté vinculado al ciclo escolar activo, cuente con semestre en curso y tenga materias asociadas. **5\. Identificar semestre en curso**  El sistema consulta la configuración académica vigente e identifica el semestre activo o en curso para el alumno seleccionado. **6\. Cargar materias y estructura bimestral**  El sistema consulta las materias asociadas al alumno para el semestre en curso y muestra la estructura de captura por materia y bimestre. **7\. Seleccionar materia y bimestre**  El actor selecciona la materia y el bimestre que desea registrar. El sistema valida que la materia corresponda al semestre del alumno y que el bimestre pertenezca al semestre en curso. **8\. Verificar calificación existente**  El sistema consulta si ya existe una calificación registrada para el alumno, materia, bimestre, semestre y ciclo escolar activo. **9\. Capturar calificación**  El actor ingresa la calificación numérica correspondiente a la materia y bimestre seleccionado. **10\. Validar calificación capturada**  El sistema valida que la calificación tenga formato numérico, que el campo no esté vacío y que no exista duplicidad para el mismo alumno, materia, bimestre, semestre y ciclo escolar. **11\. Confirmar registro**  El actor selecciona **Guardar calificación**. El sistema muestra una confirmación indicando que la calificación será almacenada en el expediente académico del alumno. **12\. Guardar calificación**  El actor confirma la operación. El sistema registra la calificación en la base de datos, vinculándola con alumno, materia, bimestre, semestre en curso, ciclo escolar activo y usuario responsable. **13\. Actualizar expediente académico**  El sistema actualiza el expediente académico del alumno y deja la información disponible para consulta, historial académico y generación de boleta. **14\. Registrar auditoría**  El sistema registra la acción en bitácora, incluyendo usuario, fecha, hora, expediente afectado, materia, bimestre, semestre y tipo de operación realizada. **15\. Mostrar confirmación final**  El sistema informa que la calificación de bachillerato fue registrada correctamente. |

El sistema no encuentra expedientes coincidentes con el criterio de búsqueda.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra el mensaje: “No se encontraron alumnos con los datos ingresados”.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor puede modificar el criterio de búsqueda o cancelar el proceso.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. Si modifica el criterio, el flujo regresa al paso<br>&nbsp;&nbsp;&nbsp;&nbsp;2. <br>&nbsp;&nbsp;&nbsp;&nbsp;5. Si cancela, el caso de uso termina sin cambios.  **A2. Ya existe calificación registrada Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;8. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema detecta que ya existe una calificación para el alumno, materia, bimestre, semestre y ciclo escolar activo.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra la calificación existente en modo consulta.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema informa que la calificación no puede sobrescribirse directamente.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema indica que cualquier cambio debe realizarse mediante el caso de uso **Modificar calificación registrada**, capturando el motivo correspondiente.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El caso de uso termina sin modificar datos.  **A3. Calificación inválida Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;10. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema detecta que la calificación ingresada contiene letras, símbolos no permitidos, formato incorrecto o se encuentra vacía.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema resalta el campo inválido.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema muestra el mensaje: “La calificación debe ser numérica”.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El actor corrige la calificación.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El sistema vuelve a validar el dato.<br>&nbsp;&nbsp;&nbsp;&nbsp;6. El flujo continúa desde el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;9. **A4. El actor cancela antes de guardar Ocurre entre los pasos 9 y<br>&nbsp;&nbsp;&nbsp;&nbsp;11. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El actor selecciona **Cancelar**.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra el mensaje: “¿Desea cancelar el registro• Los datos capturados no guardados se perderán”.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. Si el actor confirma, el sistema descarta los datos temporales.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema regresa a la pantalla anterior.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El caso de uso termina sin guardar información.  **A5. El actor no confirma el guardado Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;11. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema solicita confirmación para guardar la calificación.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El actor selecciona **No** o **Cancelar**.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema no almacena información.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El sistema conserva los datos en pantalla para edición o cancelación.<br>&nbsp;&nbsp;&nbsp;&nbsp;5. El flujo regresa al paso<br>&nbsp;&nbsp;&nbsp;&nbsp;9.  |
| **Precondiciones** | El actor debe haber iniciado sesión en el sistema. El actor debe contar con permisos para registrar calificaciones. El alumno debe estar registrado en el sistema. El alumno debe pertenecer al nivel educativo bachillerato. Debe existir un ciclo escolar activo. Debe existir un semestre en curso. El alumno debe tener materias asociadas a su grado, grupo o semestre. El expediente académico del alumno debe estar disponible. |
| **Postcondiciones** | Las calificaciones quedan registradas por alumno, materia, bimestre, semestre y ciclo escolar activo. El expediente académico del alumno queda actualizado. Las calificaciones quedan disponibles para consulta, historial académico y generación de boleta. La operación queda registrada en bitácora de auditoría cuando se genera un nuevo registro académico. |
| **Reglas de negocio involucradas** | 1\.      Solo alumnos de bachillerato pueden registrarse mediante este caso de uso. 2\.      Las calificaciones de bachillerato se registran por materia y bimestre. 3\.      El bimestre debe pertenecer al semestre en curso. 4\.      Cada calificación debe vincularse al perfil del alumno y al ciclo escolar activo. 5\.      No debe duplicarse una calificación para el mismo alumno, materia, bimestre, semestre y ciclo escolar. 6\.      Las calificaciones ya registradas no deben sobrescribirse directamente. 7\.      Las modificaciones deben realizarse mediante el caso de uso correspondiente y con motivo de modificación. 8\.      Las calificaciones deben quedar disponibles para consulta, historial académico y boleta. |
