# RF-21: Modificar datos de padre o tutor

| Campo | Descripción |
| :---- | :---- |
| **Nombre** | RF-21: Modificar datos de padre o tutor |
| **Actor** | Administrador y Gestor Administrativo |
| **Objetivo** | Actualizar la información de contacto o los datos de perfil de un padre o tutor ya registrado, reflejando los cambios de forma inmediata sin afectar el historial de facturas previas. |
| **Flujo Principal** | 1. El actor ingresa al módulo de Gestión de Padres/Tutores.<br>2. Localiza al tutor por nombre o RFC.<br>3. El sistema despliega el perfil con la información actual.<br>4. El actor hace clic en "Editar datos".<br>5. El sistema habilita el formulario con los datos precargados.<br>6. El actor actualiza los datos deseados: contacto (teléfono, correo, dirección) o perfil general (RFC, régimen fiscal, CURP del alumno, RVOE del nivel).<br>7. El actor hace clic en "Guardar cambios".<br>8. El sistema valida campos obligatorios y unicidad del RFC.<br>9. El sistema actualiza el perfil, registra en bitácora y muestra: "Datos actualizados. Los cambios fiscales aplican solo a facturas futuras". |
| **Flujo Alterno** | <b>A.</b> Cancelación:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 6, el actor hace clic en "Cancelar".<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema descarta los cambios y regresa al perfil original.<br><br><b>B.</b> RFC inválido o duplicado:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 8, el RFC tiene formato inválido o pertenece a otro tutor.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra: "El RFC ingresado no es válido o ya está registrado".<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El actor corrige y reintenta.<br><br><b>C.</b> Error de conexión:<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 9, se pierde la conexión con la base de datos.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema muestra: "Los cambios no pudieron guardarse. Verifique la conexión". |
| **Precondiciones** | • El actor debe tener sesión activa con privilegios de Administrador o Gestor Administrativo. • El perfil del tutor debe existir previamente en el sistema. |
| **Postcondiciones** | • El perfil queda actualizado y visible en todos los módulos de forma inmediata. • Los cambios fiscales aplican únicamente a facturas futuras; las anteriores no se modifican. • La acción queda registrada en la bitácora de auditoría (RF-08). |
| **Reglas de negocio involucradas** | • La modificación está habilitada para Administrador y Gestor Administrativo.<br>• Los cambios en RFC y régimen fiscal aplican solo a facturas futuras (no retroactivos).<br>• Los campos CURP del alumno y RVOE del nivel son parte del perfil general, no exclusivamente fiscales.<br>• Cualquier modificación queda registrada en la bitácora de auditoría (RF-08). |
