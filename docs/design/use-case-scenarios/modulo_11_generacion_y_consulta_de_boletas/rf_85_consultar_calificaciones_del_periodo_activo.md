# RF-85: Consultar calificaciones del periodo activo

| Campo | Descripción |
| :---: | :---- |
| **Autor** | Jessica Pablo Hernández |
| **Nombre** | RF-85: Consultar calificaciones del periodo activo |
| **Actor** | Docente |
| **Objetivo** | Permitir la consulta de las calificaciones académicas vigentes de un alumno durante el periodo activo, restringiendo la visualización exclusivamente a información académica autorizada y relevante para el ciclo actual. |
| **Flujo Principal** | **1\. Acceder al módulo de consulta académica**  El actor autenticado selecciona el módulo **Consulta de Calificaciones** desde el menú principal. El sistema valida que el usuario tenga permisos para consultar información académica. **2\. Buscar y seleccionar expediente del alumno**  El actor busca al alumno por nombre, matrícula, grupo o grado. El sistema consulta la base de datos, muestra los expedientes coincidentes y permite seleccionar el expediente correspondiente. **3\. Validar expediente académico**  El sistema verifica que el alumno exista, que el expediente académico esté disponible y que el alumno se encuentre asociado al ciclo escolar activo. **4\. Identificar periodo académico activo**  El sistema consulta la configuración académica vigente y determina automáticamente el periodo académico activo correspondiente al ciclo escolar actual. **5\. Recuperar calificaciones vigentes**  El sistema consulta las calificaciones registradas del alumno correspondientes únicamente al periodo académico activo, incluyendo materias curriculares, evaluaciones cualitativas y actividades extracurriculares autorizadas. **6\. Filtrar información restringida**  El sistema excluye automáticamente: ·         información financiera, ·         datos fiscales, ·         historial académico de ciclos anteriores, ·         documentos administrativos no autorizados. **7\. Organizar información académica**  El sistema organiza las calificaciones por materia, evaluación o actividad académica correspondiente, mostrando únicamente la información permitida para consulta. El sistema aplica automáticamente la estructura de visualización correspondiente al nivel educativo del alumno. **8\. Mostrar resultados de consulta**  El sistema presenta en pantalla las calificaciones del periodo activo, incluyendo datos académicos relevantes como materia, bloque, trimestre, bimestre o evaluación correspondiente. **9\. Permitir navegación o filtrado adicional**  El actor puede consultar otra materia, cambiar el criterio de búsqueda o regresar al listado general de alumnos sin abandonar el módulo. **10\. Registrar auditoría de consulta**  Si la política institucional lo requiere, el sistema registra en bitácora la consulta realizada, incluyendo usuario, fecha, hora y expediente consultado. **11\. Finalizar consulta**  El caso de uso termina cuando el actor concluye la revisión de las calificaciones del periodo activo o abandona el módulo. |

El sistema no encuentra un periodo académico activo configurado.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El sistema bloquea la consulta.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El sistema muestra el mensaje: “No existe un periodo académico activo para realizar la consulta”.<br>&nbsp;&nbsp;&nbsp;&nbsp;4. El caso de uso termina. **A3. El alumno no tiene calificaciones registradas en el periodo activo Ocurre en el paso<br>&nbsp;&nbsp;&nbsp;&nbsp;5. **<br>&nbsp;&nbsp;&nbsp;&nbsp;1. El sistema no encuentra calificaciones registradas para el periodo activo.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. | **Precondiciones** | El actor debe haber iniciado sesión en el sistema. El actor debe contar con permisos para consultar información académica. Debe existir un ciclo escolar activo. Debe existir un periodo académico activo. El alumno debe estar registrado en el sistema. El expediente académico del alumno debe estar disponible. |
| **Postcondiciones** | El actor visualiza únicamente las calificaciones correspondientes al periodo académico activo. No se muestran datos financieros, fiscales ni históricos de otros ciclos escolares. La consulta queda registrada en bitácora si la política institucional requiere auditoría de accesos académicos. No se modifica información académica. |
| **Reglas de negocio involucradas** | Solo deben mostrarse calificaciones del periodo académico activo. El módulo no debe mostrar información financiera ni fiscal. El módulo no debe mostrar historiales académicos de ciclos anteriores. La información académica debe filtrarse según permisos del usuario. Las consultas no deben modificar registros académicos. Las consultas pueden registrarse en bitácora según políticas institucionales. Solo usuarios autorizados pueden acceder al módulo. |
